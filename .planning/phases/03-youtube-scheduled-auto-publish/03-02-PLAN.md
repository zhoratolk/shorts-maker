<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local script → YouTube Data API | Every upload/schedule call crosses to Google over the network; the request body (privacyStatus/publishAt) must be exactly right or a video publishes at the wrong time/visibility. |
| OAuth token file → filesystem | upload_token.json is a bearer credential at rest on local disk; leakage lets an attacker post videos as this channel. |
| config.publish.enabled → live network egress | The gate between "computes a body in memory" (dry-run) and "actually uploads" (live). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-04 | Information Disclosure | upload_token.json at rest | high | mitigate | Gitignored (Task 3) + scope minimization: the token requests ONLY youtube.upload (never plain `youtube`/`youtubepartner`), so a leak can post-private-scheduled but cannot read analytics/channel data the read-only token can (D-08/D-09). Build-time check: grep the scopes=[...] at the call site is exactly [UPLOAD_SCOPE]. |
| T-03-05 | Elevation of Privilege | publish.enabled dry-run gate | high | mitigate | The enabled check is the first statement in upload_and_schedule, before service_factory/credential load — unit-tested that dry-run performs zero calls, so a live upload cannot happen without explicit opt-in (PUB-03). |
| T-03-06 | Tampering | build_insert_body publishAt | medium | mitigate | publishAt sourced only from next_free_slot's future-guarded UTC math (Pitfall 1); privacyStatus hardcoded "private" (never derived from mutable input) so a bad manifest field can't force an immediate public post. Field-length validation fails the item loud rather than submitting malformed data. |
| T-03-07 | Repudiation / duplicate post | write-ahead status=uploading | high | mitigate | Manifest records UPLOADING (persisted) before videos.insert; a crash between persist and .execute() leaves a durable trace that Plan 03 reconciles instead of blind re-upload (PUB-05). |
| T-03-SC | Tampering | package installs | high | mitigate | No new packages introduced (03-RESEARCH Package Legitimacy Audit: unchanged package set — googleapiclient/google-auth-* already installed and previously audited). No install step in this phase. |
</threat_model>
