## Phase Goal

**As a** solo creator, **I want to** be warned when a clip contains likely-licensed music detected by local audio fingerprinting, **so that** I can catch a probable copyright/Content-ID risk before publishing — without the check ever breaking my pipeline when the fingerprint tool is unavailable.

<objective>
Deliver the audio-fingerprinting tier of monetization-risk flagging (MONET-01): a local Chromaprint/pyacoustid step that fingerprints a clip's audio, flags likely-copyrighted audio as an advisory risk, and merges that flag into the per-platform risk-dict shape established by Plan 01 (MONET-03 — confidence-carrying, advisory, never blocking). The external `fpcalc` binary is added to setup.py's dependency-check pattern (like ffmpeg), and the whole step is fail-open: no binary, no network, or no match all degrade to "no copyright flag this run" rather than aborting.

Purpose: Completes MONET-01, the last uncovered monetization requirement. Sequenced after Plan 01 because it merges into Plan 01's risk-dict shape and shares config.py / config.example.yaml (file-level dependency → Wave 2). Kept as its own plan because it introduces a new external binary dependency and a distinct subsystem (audio fingerprinting) separate from the text-rule scorer.

Output: `scripts/monetization_audio.py`, a `check_fpcalc()` in setup.py, an `audio_fingerprint` sub-config, and unit coverage — with the copyright flag flowing into the same metadata risk block Plan 01 built.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| clip audio → fpcalc subprocess | local binary invocation; failure must fail open |
| fingerprint → AcoustID lookup (optional) | the ONLY network egress in this phase; off by default, fail-open when enabled |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-04 | Denial of service | missing/broken fpcalc binary | low | mitigate | shutil.which guard + FileNotFoundError fail-open → warn + None, pipeline continues (CONVENTIONS fail-open) |
| T-01-05 | Information disclosure | AcoustID network lookup sends an audio fingerprint off-device | medium | mitigate | Lookup is opt-in (enable_lookup=False default); only a derived fingerprint (not raw audio) is sent; fail-open on network error preserves local-first default |
| T-01-06 | Spoofing | fingerprint false-positive treated as certain copyright | medium | mitigate | Flag framed as advisory "likely licensed audio" with a confidence, never a gate (PITFALL 2) |
</threat_model>
