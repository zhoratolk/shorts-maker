## Phase Goal

**As a** solo creator, **I want to** derive a structured style profile of concrete few-shot naming and moment-selection examples from my own upload history, **so that** later phases can generate titles in my real voice — without any raw channel data ever landing in a committed file.

<objective>
Deliver the creator style-profile step (STYLE-01, STYLE-02, STYLE-03) end-to-end: read the creator's real upload history from the existing `scripts/youtube_analytics.py` cache (STYLE-01, reusing its OAuth/cache pattern — no parallel auth flow), derive concrete few-shot naming/moment-selection examples as structured data rather than prose summaries (STYLE-02), and write the result ONLY to a gitignored cache under `work/_profile/style_profile.json` (STYLE-03), matching the same privacy discipline as the analytics cache.

Purpose: This is the second, fully-independent half of Phase 1 (parallel to MONET). It produces the artifact Phase 2's LLM titling consumes as few-shot grounding. The privacy constraint is load-bearing: this project had a real leaked-channel-data incident (PITFALL 4), so the output must be structurally gitignored and verified clean via `git status` — never written into a tracked docs/ file the way the original incident happened.

Output: `scripts/style_profile.py` (a pure function over the already-cached analytics JSON, no network of its own), its test, and a `.gitignore` entry for `work/_profile/` (belt-and-suspenders — `work/` is already ignored, but an explicit entry documents the privacy boundary).
</objective>
