<objective>
Complete `scripts/transitions.py` with the decision layer on top of 04-03's signal functions: a pure `classify_transition` that maps the three boundary scores to one of the 6 transition types with a conservative bias (D-01), an adaptive per-video threshold helper (mirroring audio_energy.py's relative-baseline approach, per D-02 and 04-RESEARCH.md Pitfall 4), a `select_boundary_transitions` orchestration function that scores every boundary of a clip and returns the per-boundary type list (fail-open to all-'cut' when analysis is unavailable or the borrowable gap is too small), and the `select-transitions` CLI subcommand the orchestrator (04-06) calls.

Purpose: TRANS-02 (all 6 types selectable) and TRANS-03 (inconclusive/insufficient-overlap → cut fallback). Per the downstream note, select_boundary_transitions is written to be generic over any keep_segments list so Phase 5's cross-clip compilation can reuse it, not hard-coded to the jumpcut-splice case.
Output: classifier + adaptive threshold + boundary-selection orchestration + CLI in scripts/transitions.py, with unit tests.
