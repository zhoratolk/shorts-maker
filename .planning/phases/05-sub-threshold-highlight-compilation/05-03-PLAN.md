<objective>
Add `build_compilation_command` to `scripts/render.py`: a new ffmpeg command builder that opens the source video once per compilation member (`-ss`/`-i`/`-t` per member, never one decode window spanning the whole compilation), flattens every member's own segments (its internal jump cuts if any, else its single `start`/`end` window) into one render-order list, folds them pairwise via the same `build_transition_filter`/concat-downgrade machinery `_build_transition_fold` already uses for single-clip jump cuts — but capping any real-transition overlap to a small fraction of each side's own duration instead of borrowing a (nonexistent, for separately-approved candidates) pause gap — then applies crop/punch-zoom/subtitles/fade exactly once on the folded result (D-06). Wire `render_clip` to dispatch to it whenever a PLAN.json entry has `"type": "compilation"`, without disturbing the existing single-clip path.

Output: extended `scripts/render.py`, extended `tests/test_render.py` and `tests/test_integration_ffmpeg.py`.
