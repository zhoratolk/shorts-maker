    - `class CompilationError(ValueError): pass`. `MIN_GROUP_SIZE = 2` module constant.
    - `build_compilation_entry(members: list[dict], compilation_max_seconds: float, crop_style: str, boundary_transitions: list[str] | None = None, punch_zoom_at: float | None = None, subtitles_path: str | None = None, metadata_path: str | None = None, output_filename: str | None = None) -> dict`.
    - `members` is caller-supplied, already ordered strongest-first (D-04). Each member dict carries `video_stem` (str — which source video/session it came from, COMP-03's mechanical guard target), `start` (float, absolute source seconds), `end` (float), and optionally `keep_segments` (list of `[float, float]` absolute-second pairs, present only when this member itself had internal jump cuts computed in step 5 before it was found sub-threshold).
    - Guard 1 (COMP-02): if `len(members) < MIN_GROUP_SIZE`, raise `CompilationError` naming the actual count — "a compilation group needs >= {MIN_GROUP_SIZE} members, got {N}".
    - Guard 2 (COMP-03): compute `{m["video_stem"] for m in members}`; if more than one distinct value, raise `CompilationError` listing the distinct stems — "all group members must share one video_stem, got {sorted list}". This is a defense-in-depth mechanical check even though the normal single-video pipeline run already scopes candidates.json to one video_stem — it exists for the "library-wide search" scenario (SKILL.md's documented re-run of steps 2-5 across a subset of already-transcribed videos) where a grouping mistake could otherwise pool candidates from two different source videos.
    - Length-ceiling capping (D-04/D-05): iterate `members` in the given (strongest-first) order, accumulating each member's own duration — `sum(seg_end - seg_start for seg_start, seg_end in member["keep_segments"])` if `keep_segments` is present, else `member["end"] - member["start"]` — and keep appending members to a `fitted` list only while the running total stays `<= compilation_max_seconds`. Stop at the first member that would push the total over the cap (drop it and every member after it in this call — do not skip ahead looking for a smaller later member that might still fit; strongest-first order must be preserved literally). If `len(fitted) < MIN_GROUP_SIZE` after capping (e.g. the single strongest member alone already exceeds the cap), raise `CompilationError` — never return a single-member "compilation". Never raise merely because capping shrank a valid (>=2-member) group; that is expected, working behavior, not an error.
    - Compute the total flattened segment count across `fitted` members: `sum(len(m["keep_segments"]) if m.get("keep_segments") else 1 for m in fitted)`. If `boundary_transitions` is not None, its length must equal `flattened_segment_count - 1` — raise `CompilationError` naming both numbers otherwise. This mirrors render.py's own boundary_transitions length guard (Plan 05-03) but fails earlier/mechanically, before the entry ever reaches render.py — both checks are intentional defense-in-depth, not redundant dead code (PLAN.json is a file-based hand-off artifact a corrupted write could still bypass one of the two).
    - Build and return the entry: `{"type": "compilation", "segments": [{"start": m["start"], "end": m["end"], **({"keep_segments": m["keep_segments"]} if m.get("keep_segments") else {})} for m in fitted], "crop_style": crop_style}` plus `boundary_transitions`/`punch_zoom_at`/`subtitles_path`/`metadata_path`/`output_filename` each included only when their parameter is not `None` (never `null`/`false` placeholders — matches the project's existing optional-field-omitted convention already used for single-clip PLAN.json entries).
    - CLI wrapper (project convention: every `scripts/*.py` module pairs its Python API with an `argparse` `main()`, per CLAUDE.md Naming Patterns): a single non-subcommand `main()` (mirroring `scripts/naming.py`'s plain-positional-args shape, since compilation.py has only one operation) taking `members_json` (path to a JSON list matching the `members` shape above), `compilation_max_seconds` (float), `crop_style`, `output_json` (path to write the built entry to) as positionals, plus `--boundary-transitions-json`, `--punch-zoom-at` (float), `--subtitles-path`, `--metadata-path`, `--output-filename` as optional flags defaulting to `None`.
