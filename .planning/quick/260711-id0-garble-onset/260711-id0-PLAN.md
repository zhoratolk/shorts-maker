<objective>
Add two profanity-masking refinements to shorts-maker's Phase 7 pipeline, extending the existing duck+garble mask (07-03/07-04):

1. **Onset delay** — let a mask start a fraction of a second *after* the word begins, so the word's leading transient plays clean and the rest is masked. Empirically ≤ ~0.12s keeps STT-defeat intact; larger onsets let faster-whisper recover the word from its onset + context.
2. **Custom sound-censor mode** — replace a profane word's audio with a user-supplied censor clip (mute the main track inside the span, overlay the censor sound via `amix`) as an alternative to the garble mask.

All technical decisions are ALREADY empirically validated (work/_profanity_demo/, faster-whisper STT re-checks). This plan FIXES those validated decisions into code — do not re-research or re-tune. In particular: do NOT change the shipped garble defaults (garble_freq 1800, width 4, warble_freq 18, depth 0.7) — Plan 07-03 has exact-value unit tests on them.

Purpose: give the operator a second, cleaner-sounding censor option and a knob to keep more of the word audible without leaking it to platform STT scanners.
Output: config schema + validation, onset-shifted span detection, sound-censor render path, and full unit coverage — with defaults preserved byte-identically.
</objective>
