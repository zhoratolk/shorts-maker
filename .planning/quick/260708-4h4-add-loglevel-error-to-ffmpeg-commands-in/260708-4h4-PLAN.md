`"-loglevel", "error"` to both ffmpeg argument builders in `scripts/render.py`, so that
when `render_clip` raises `RenderError` with the captured `result.stderr`, that stderr
carries only the real error message instead of build config, codec lists, and progress noise.

Purpose: shrink error output / token cost when debugging render failures. No behavior change
on success (ffmpeg is silent either way with `-loglevel error`).
Output: edited `scripts/render.py` (two builders) and `tests/test_render.py` (assertions
re-aligned for the 2-element index shift).
</objective>
