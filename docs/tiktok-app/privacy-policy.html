§1 What this is

video.publish is a private automation tool built and used by one creator to publish video clips — already recorded and edited by that same creator — to their own TikTok account. It is not distributed to other users and does not act on behalf of anyone except its operator.

§2 What we access

Through TikTok's Content Posting API, with the scopes video.publish and video.upload, the integration can:

  • Upload a video file the operator has already produced, to the operator's own account.
  • Check the publish status of that upload.

It never reads, follows, messages, or otherwise touches any other TikTok account, and it requests no scope beyond the two above.

§3 What we store

An OAuth access token and refresh token issued by TikTok, stored only on the operator's own device, used solely to authenticate the API calls in §2. No video content, caption text, or account data is copied to any third-party server, database, or analytics service — the token is the only thing persisted, and it never leaves the local machine that runs this tool.

§4 What we don't do

  • No collection of data belonging to any TikTok user other than the operator.
  • No selling, renting, or sharing of any data with third parties.
  • No advertising or third-party tracking SDKs of any kind.
  • No use of the data for anything other than the single publishing action described in §2.

§5 Retention & revocation

The stored token is kept until the operator revokes it. It can be revoked at any time from TikTok's own Settings → Manage account → Apps and websites panel, or by deleting the local token file, either of which immediately ends this integration's access.

§6 Children's data

This tool is an internal utility for one adult creator's own workflow. It is not directed at children and does not knowingly process data from anyone under 13.

§7 Changes to this policy

If this policy changes, the effective date above will be updated. Material changes will be reflected here before they take effect.
